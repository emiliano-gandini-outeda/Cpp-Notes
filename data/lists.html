A list is a collection of elements organized sequentially. Unlike arrays, lists can be dynamic, meaning their size can grow or shrink during runtime. They are extremely useful when you need flexible insertion or deletion of elements.

In C++, arrays are the simplest kind of sequential storage:

#include <iostream>
using namespace std;

int main() {
    int arr[5] = {10, 20, 30, 40, 50};
    cout << arr[2] << endl;  // Access third element
    return 0;
}
  • Fixed size â†’ cannot grow or shrink.
  • Access â†’ O(1) time using index.
  • Use case â†’ fast, static storage for known-size collections.

To overcome fixed-size limitations, C++ provides vectors (dynamic arrays):

#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<int> v = {1, 2, 3};
    v.push_back(4);          // Add at the end
    v.insert(v.begin() + 1, 5); // Insert at index 1
    v.erase(v.begin() + 2);  // Remove element at index 2

    for (int x : v) cout << x << " ";
    return 0;
}
  • Dynamic size â†’ grows or shrinks automatically.
  • Access â†’ O(1) for direct indexing, O(n) for insertion/deletion in the middle.
  • Use case â†’ collections with frequent additions/removals.

ðŸ”— Linked Lists: Flexibility in Memory

Unlike arrays or vectors, linked lists store elements as nodes, each containing data and a pointer to the next node. Nodes are not contiguous in memory, which allows efficient insertion and deletion anywhere in the list.

Singly Linked List (SLL)

  • Each node points forward only.
  • Traversal is possible in one direction.
#include <iostream>
using namespace std;

struct Node {
    int data;
    Node* next;
};

void insertAtHead(Node*& head, int value) {
    Node* newNode = new Node{value, head};
    head = newNode;
}

void printList(Node* head) {
    while(head) {
        cout << head->data << " -> ";
        head = head->next;
    }
    cout << "NULL\n";
}

int main() {
    Node* list = nullptr;
    insertAtHead(list, 10);
    insertAtHead(list, 20);
    insertAtHead(list, 30);

    printList(list);
    return 0;
}

Doubly Linked List (DLL)

  • Each node points forward and backward.
  • Traversal possible both directions.
#include <iostream>
using namespace std;

struct Node {
    int data;
    Node* next;
    Node* prev;
};

void insertAtHead(Node*& head, int value) {
    Node* newNode = new Node{value, head, nullptr};
    if(head) head->prev = newNode;
    head = newNode;
}

void printForward(Node* head) {
    while(head) {
        cout << head->data << " <-> ";
        head = head->next;
    }
    cout << "NULL\n";
}

void printBackward(Node* tail) {
    while(tail) {
        cout << tail->data << " <-> ";
        tail = tail->prev;
    }
    cout << "NULL\n";
}

int main() {
    Node* list = nullptr;
    insertAtHead(list, 10);
    insertAtHead(list, 20);
    insertAtHead(list, 30);

    printForward(list);
    return 0;
}

Circular Linked List (CLL)

  • Last node points back to the first node, forming a loop.
  • Can be singly or doubly linked.
#include <iostream>
using namespace std;

struct Node {
    int data;
    Node* next;
};

void insert(Node*& head, int value) {
    Node* newNode = new Node{value, nullptr};
    if(!head) {
        head = newNode;
        newNode->next = head;
    } else {
        Node* temp = head;
        while(temp->next != head) temp = temp->next;
        temp->next = newNode;
        newNode->next = head;
    }
}

void printList(Node* head) {
    if(!head) return;
    Node* temp = head;
    do {
        cout << temp->data << " -> ";
        temp = temp->next;
    } while(temp != head);
    cout << "(head)\n";
}

int main() {
    Node* list = nullptr;
    insert(list, 10);
    insert(list, 20);
    insert(list, 30);

    printList(list);
    return 0;
}

âœ… Key Takeaways

  • Use arrays for fixed-size data and fast random access.
  • Use vectors for dynamic-size collections with occasional middle insertions.
  • Use linked lists when frequent insertion/deletion is needed or memory is fragmented.
  • Choose circular or doubly linked lists for specialized scenarios like looping or bidirectional navigation.
