‚úÖ What is Dynamic Memory Allocation?

Dynamic memory allocation allows a C++ program to allocate memory at runtime instead of at compile-time.

  • Memory is allocated from the heap, not the stack.
  • Useful when the size of data is unknown at compile time.
  • Managed manually using the new and delete operators.

üìå How new and delete Work

  • new allocates memory and returns a pointer to it.
  • delete frees memory previously allocated by new.

  • For arrays:

    • Use new[] to allocate an array.
    • Use delete[] to deallocate the array.
  • Failing to delete memory causes memory leaks.

üßæ Syntax

Single variable 

int* ptr = new int;    // Allocate memory for one int
*ptr = 42;             // Assign value
delete ptr;            // Free memory

Array of variables 

int* arr = new int[5]; // Allocate array of 5 ints
arr[0] = 10;           // Assign values
arr[1] = 20;
delete[] arr;          // Free array memory

üßæ Example Program 

#include <iostream>
using namespace std;

int main() {
    // Allocate a single int
    int* num = new int;
    *num = 100;
    cout << "Value: " << *num << endl;
    delete num; // Free memory

    // Allocate an array
    int size = 5;
    int* numbers = new int[size];
    for (int i = 0; i < size; i++)
        numbers[i] = i * 10;

    cout << "Array values: ";
    for (int i = 0; i < size; i++)
        cout << numbers[i] << " ";
    cout << endl;

    delete[] numbers; // Free array memory

    return 0;
}

Output:

Value: 100
Array values: 0 10 20 30 40

‚ö†Ô∏è Important Rules

  1. Always use delete or delete[] to free memory allocated with new / new[].
  2. Do not mix delete and new[] (or delete[] with new).
  3. Uninitialized pointers can cause undefined behavior; always initialize or assign after allocation.
  4. Accessing memory after deletion is undefined behavior (dangling pointer).
  5. Dynamic allocation is slower than stack allocation and increases risk of memory leaks if not handled properly.
