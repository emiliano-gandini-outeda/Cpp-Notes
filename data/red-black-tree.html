‚úÖ What is a Red-Black Tree?

A Red-Black Tree is a self-balancing binary search tree where each node has a color attribute: red or black.

It ensures the tree remains roughly balanced, giving O(log n) time complexity for insertion, deletion, and search.

Properties of a Red-Black Tree:

  1. Every node is either red or black.
  2. The root is always black.
  3. All leaves (NULL nodes) are black.
  4. Red nodes cannot have red children (no two consecutive red nodes).
  5. Every path from a node to its descendant leaves contains the same number of black nodes (black-height).

These rules guarantee that the longest path is at most twice the length of the shortest path, maintaining balance.

üß† Syntax (Node Definition) 

enum Color { RED, BLACK };

struct Node {
    int data;
    Color color;
    Node* left;
    Node* right;
    Node* parent;
};
  • data: The value stored in the node.
  • color: Red or Black.
  • left, right, parent: Pointers to children and parent node.

üßæ Example Program ‚Äì Basic Insertion (Conceptual) 

#include <iostream>
using namespace std;

enum Color { RED, BLACK };

struct Node {
    int data;
    Color color;
    Node* left;
    Node* right;
    Node* parent;
};

// Create a new red node
Node* createNode(int data) {
    Node* node = new Node{data, RED, nullptr, nullptr, nullptr};
    return node;
}

// In-order traversal
void inOrder(Node* root) {
    if (root) {
        inOrder(root->left);
        cout << root->data << (root->color == RED ? "(R) " : "(B) ");
        inOrder(root->right);
    }
}

int main() {
    Node* root = createNode(10);
    root->color = BLACK; // root must be black
    Node* node1 = createNode(5);
    Node* node2 = createNode(20);

    // Conceptual linking
    root->left = node1; node1->parent = root;
    root->right = node2; node2->parent = root;

    cout << "In-order traversal: ";
    inOrder(root);  // Output: 5(R) 10(B) 20(R)
    cout << endl;

    return 0;
}
‚ö†Ô∏è Note: Real Red-Black insertion involves rotations and recoloring to maintain all properties. This example shows the basic structure and traversal.

‚ö†Ô∏è Important Rules

  1. Root is always black.
  2. No two consecutive red nodes.
  3. All paths from node to leaves must have the same number of black nodes.
  4. Insertion/deletion may require rotations and recoloring to maintain properties.
  5. Provides guaranteed O(log n) operations, even in worst case.
