üìö Basic Data Structures

Basic data structures form the foundation for algorithms and efficient data management. Mastering them is key to designing performant software. The main structures include arrays, lists, stacks, queues, linked lists, and hash tables.

üü¶ Arrays & Lists

  • Array: A collection of elements of the same type stored contiguously in memory. Arrays allow direct access to any element in constant time (O(1)) but have a fixed size.
  • List (Dynamic Array / Vector): A flexible structure that can grow or shrink at runtime. Insertion or deletion may involve moving elements, making some operations slower than arrays.
  • Key idea: Arrays = fixed and fast access; Lists = dynamic and flexible.

üìö Stacks

  • Stack: Follows LIFO (Last In, First Out). The last element added is the first to be removed.
  • Typical uses: recursion, undo features, expression evaluation.

  • Main operations:

    • push ‚Üí add element on top
    • pop ‚Üí remove top element
    • top ‚Üí access top element without removing it

üü© Queues

  • Queue: Follows FIFO (First In, First Out). The first element added is the first to be removed.
  • Typical uses: process scheduling, print queues, BFS (Breadth-First Search).

  • Main operations:

    • enqueue ‚Üí add element at the end
    • dequeue ‚Üí remove element from the front
    • front ‚Üí access first element
    • back ‚Üí access last element

üîó Linked Lists

Linked lists are made of nodes containing a value and one or more pointers. They allow flexible memory allocation and easy insertions/deletions without shifting large blocks of data.

  • Singly Linked List: Nodes point only forward; simple traversal.
  • Doubly Linked List: Nodes point both forward and backward; supports bidirectional traversal.
  • Circular Linked List: Last node points back to the first, forming a loop; can be singly or doubly linked.

üóùÔ∏è Hash Tables / Maps

  • Hash Table: Stores key-value pairs for extremely fast access using a hash function that maps keys to indices.

  • Key concepts:

    • Hash function: transforms keys into array indices.
    • Collisions: when multiple keys map to the same index; handled via chaining (linked lists) or open addressing.
    • Load factor: ratio of stored elements to table size; higher values increase collisions.
  • Common uses: configuration maps, frequency counters, caches, key-value databases.
