‚úÖ What is a Priority Queue?

A priority queue is an abstract data type similar to a queue, but each element has a priority associated with it.

  • Elements with higher priority are removed before elements with lower priority.
  • Often implemented using a heap (max-heap or min-heap).

Use Cases:

  • Task scheduling
  • Dijkstra‚Äôs shortest path algorithm
  • Huffman coding
  • Real-time simulations

üß† Key Concepts

Concept Description
Element Item stored in the queue
Priority Value that determines the order of removal
Max-priority queue Highest priority element is removed first
Min-priority queue Lowest priority element is removed first
Heap Common underlying data structure for efficient implementation

üßæ Example ‚Äì Using std::priority_queue in C++ 

#include <iostream>
#include <queue>
using namespace std;

int main() {
    // Max-priority queue (default)
    priority_queue<int> pq;

    pq.push(10);
    pq.push(5);
    pq.push(20);
    pq.push(15);

    cout << "Elements in priority order: ";
    while (!pq.empty()) {
        cout << pq.top() << " ";  // Always prints the largest element
        pq.pop();
    }
    cout << endl;

    return 0;
}

Output:

20 15 10 5

üîÑ Min-Priority Queue Example 

#include <iostream>
#include <queue>
#include <vector>
using namespace std;

int main() {
    // Min-priority queue
    priority_queue<int, vector<int>, greater<int>> minPQ;

    minPQ.push(10);
    minPQ.push(5);
    minPQ.push(20);
    minPQ.push(15);

    cout << "Elements in priority order: ";
    while (!minPQ.empty()) {
        cout << minPQ.top() << " ";  // Always prints the smallest element
        minPQ.pop();
    }
    cout << endl;

    return 0;
}

Output:

5 10 15 20

‚ö†Ô∏è Important Notes

  1. Default std::priority_queue is a max-heap.
  2. Min-heap can be implemented using greater<T> comparator.
  3. Common operations:
    • push(x) ‚Üí insert element
    • top() ‚Üí get element with highest priority
    • pop() ‚Üí remove element with highest priority
    • empty() ‚Üí check if queue is empty
  4. Time complexity:
    • push ‚Üí O(log n)
    • pop ‚Üí O(log n)
    • top ‚Üí O(1)
