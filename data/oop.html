‚úÖ What is Object-Oriented Programming?

Object-Oriented Programming (OOP) is a programming paradigm that organizes software around objects instead of functions or procedures.

  • Objects are instances of classes that encapsulate data (attributes) and behavior (methods/functions).
  • OOP aims to model real-world entities in code, making programs more modular, reusable, and maintainable.
  • The main concepts of OOP are Classes, Objects, Encapsulation, Inheritance, Abstraction, and Polymorphism.

üìå Classes and Objects

Class:

  • Defines data members (attributes) and member functions (methods).
  • Acts as a blueprint for creating multiple objects with the same structure and behavior.

Syntax Example:

#include <iostream>
using namespace std;

class Car {
public:
    string brand;     // Attribute
    string model;
    int year;

    void startEngine() { // Method
        cout << "Engine started for " << brand << " " << model << endl;
    }
};

Object:

  • An instance of a class.
  • Allocated either statically (on stack) or dynamically (on heap).

Example:

int main() {
    // Static allocation
    Car car1;
    car1.brand = "Toyota";
    car1.model = "Corolla";
    car1.year = 2020;

    car1.startEngine(); // Call method

    // Dynamic allocation
    Car* car2 = new Car();
    car2->brand = "Honda";
    car2->model = "Civic";
    car2->year = 2022;

    car2->startEngine();

    delete car2; // Free dynamic memory
    return 0;
}

Output:

Engine started for Toyota Corolla
Engine started for Honda Civic

üßæ Constructors and Destructors

Constructor:

  • A special function with the same name as the class.
  • Automatically called when an object is created.
  • Can have parameters to initialize attributes. 
class Car {
public:
    string brand;
    string model;
    int year;

    // Constructor
    Car(string b, string m, int y) {
        brand = b;
        model = m;
        year = y;
    }

    void startEngine() {
        cout << "Engine started for " << brand << " " << model << endl;
    }
};

int main() {
    Car car1("Ford", "Mustang", 2021);
    car1.startEngine();
}

Destructor:

  • Special function with a ~ prefix, called when the object is destroyed.
  • Useful for cleaning up resources, especially with dynamic memory. 
class Car {
public:
    Car() { cout << "Constructor called!" << endl; }
    ~Car() { cout << "Destructor called!" << endl; }
};

üîÑ Member Access Specifiers

Specifier Access
public Accessible from outside the class
private Accessible only inside the class
protected Accessible inside class and derived classes

Example:

class Car {
private:
    int engineNumber;  // Cannot access outside

public:
    string brand;

    void setEngineNumber(int num) {
        engineNumber = num;
    }

    int getEngineNumber() {
        return engineNumber;
    }
};

‚ö†Ô∏è Important Rules

  1. Objects have unique identity and state, even if created from the same class.
  2. Prefer constructors to initialize objects instead of setting attributes manually.
  3. Always delete dynamically allocated objects to prevent memory leaks.
  4. Keep attributes private and access them via getter/setter methods to maintain encapsulation.
  5. Methods can be defined inside or outside the class. If outside, use the scope resolution operator ::.

‚úÖ What is Abstraction?

Abstraction in OOP is the process of hiding implementation details and exposing only the essential features of a class or object.

  • Focuses on what an object does, not how it does it.
  • Helps reduce complexity by providing a simplified interface.
  • Supports modularity, maintainability, and reusability of code.

Key idea: Users of a class should know what operations are available, but not how they are implemented internally.

üß† Key Concepts of Abstraction

Concept Description
Interface The exposed set of methods and properties a class provides
Implementation Hidden details inside the class that perform the actual work
Abstract class A class that cannot be instantiated directly and may contain pure virtual functions
Pure virtual function A function declared with = 0, requiring derived classes to implement it
Encapsulation vs Abstraction Encapsulation hides data, abstraction hides implementation details
Benefits Simplifies usage, reduces complexity, supports maintainability

üìå Implementing Abstraction in C++

Using Abstract Classes and Pure Virtual Functions

  • A pure virtual function is a function without a body in the base class, forcing derived classes to implement it.
  • A class containing at least one pure virtual function becomes an abstract class, which cannot be instantiated.

Syntax Example:

#include <iostream>
using namespace std;

// Abstract base class
class Shape {
public:
    virtual double area() = 0; // Pure virtual function
    virtual double perimeter() = 0;
};

class Rectangle : public Shape {
private:
    double width, height;
public:
    Rectangle(double w, double h) : width(w), height(h) {}
    double area() override { return width * height; }
    double perimeter() override { return 2 * (width + height); }
};

class Circle : public Shape {
private:
    double radius;
public:
    Circle(double r) : radius(r) {}
    double area() override { return 3.14159 * radius * radius; }
    double perimeter() override { return 2 * 3.14159 * radius; }
};

int main() {
    Shape* s1 = new Rectangle(5, 3);
    Shape* s2 = new Circle(4);

    cout << "Rectangle area: " << s1->area() << endl;
    cout << "Rectangle perimeter: " << s1->perimeter() << endl;

    cout << "Circle area: " << s2->area() << endl;
    cout << "Circle perimeter: " << s2->perimeter() << endl;

    delete s1;
    delete s2;
    return 0;
}

Output:

Rectangle area: 15
Rectangle perimeter: 16
Circle area: 50.2654
Circle perimeter: 25.1327

üîÑ How Abstraction Works in the Example

  1. The Shape class defines what methods a shape must have (area() and perimeter()), without specifying how they work.
  2. Derived classes (Rectangle and Circle) implement the actual calculations.
  3. Code that uses Shape* pointers can work without knowing the type of shape or its implementation details.
  4. Users interact with the interface (methods) only, ignoring internal complexity.

‚ö†Ô∏è Important Rules

  1. Abstract classes cannot be instantiated directly: Shape s; ‚Üí ‚ùå
  2. Derived classes must implement all pure virtual functions or remain abstract.
  3. Virtual functions allow dynamic dispatch, enabling polymorphic behavior (covered later).
  4. Abstraction helps decouple interface from implementation, making code easier to maintain and extend.

‚úÖ What is Polymorphism?

Polymorphism is the ability of a single function, operator, or object to take many forms.

  • In C++, it allows different classes to define the same interface and behave differently.
  • Polymorphism is essential for flexible, reusable, and extensible code.

  • Two main types in C++:

    1. Compile-time (Static) Polymorphism
    2. Run-time (Dynamic) Polymorphism

üß† Key Concepts of Polymorphism

Concept Description
Function Overloading Same function name, different parameters (compile-time)
Operator Overloading Define behavior of operators for user-defined types
Virtual Functions Allow derived class methods to override base class methods (run-time)
Overriding Redefining a base class function in a derived class
Dynamic Dispatch Choosing which function to call at runtime via vtable
Benefits Code flexibility, interface consistency, modularity, maintainability

üìå Compile-time (Static) Polymorphism

Occurs when the decision about which function to call is made at compile time.

Function Overloading 

#include <iostream>
using namespace std;

class Calculator {
public:
    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }
};

int main() {
    Calculator calc;
    cout << "Sum int: " << calc.add(2, 3) << endl;
    cout << "Sum double: " << calc.add(2.5, 3.5) << endl;
}

Output:

Sum int: 5
Sum double: 6

Operator Overloading 

#include <iostream>
using namespace std;

class Vector {
public:
    int x, y;
    Vector(int a, int b) : x(a), y(b) {}

    // Overload + operator
    Vector operator+(const Vector& v) {
        return Vector(x + v.x, y + v.y);
    }
};

int main() {
    Vector v1(1, 2), v2(3, 4);
    Vector v3 = v1 + v2;
    cout << "v3: (" << v3.x << "," << v3.y << ")" << endl;
}

Output:

v3: (4,6)

üìå Run-time (Dynamic) Polymorphism

Occurs when the decision about which function to call is made at runtime, usually via virtual functions and inheritance.

Virtual Functions 

#include <iostream>
using namespace std;

class Animal {
public:
    virtual void makeSound() {  // Virtual function
        cout << "Some generic animal sound" << endl;
    }
};

class Dog : public Animal {
public:
    void makeSound() override {
        cout << "Woof Woof" << endl;
    }
};

class Cat : public Animal {
public:
    void makeSound() override {
        cout << "Meow" << endl;
    }
};

int main() {
    Animal* a1 = new Dog();
    Animal* a2 = new Cat();

    a1->makeSound(); // Calls Dog::makeSound()
    a2->makeSound(); // Calls Cat::makeSound()

    delete a1;
    delete a2;
}

Output:

Woof Woof
Meow
  • The base class declares makeSound() as virtual.
  • At runtime, C++ uses the vtable to call the correct derived class method.
  • This is dynamic polymorphism, enabling flexible and extensible code.

‚ö†Ô∏è Important Rules

  1. Use the keyword virtual in the base class for methods you want to override.
  2. Derived classes can override the base method with the same signature.
  3. To force derived classes to implement a method, make it pure virtual: 
    virtual void makeSound() = 0;
  4. Dynamic polymorphism works only through pointers or references to base class.
  5. Overloaded operators/functions are resolved at compile-time, whereas virtual functions are resolved at runtime.

‚úÖ Summary Table

Concept Example / Notes
Function overloading Same name, different parameters
Operator overloading Custom behavior for operators
Virtual function virtual void func() in base class
Overriding Redefine base class function in derived class
Compile-time polymorphism Function/operator overloading
Run-time polymorphism Virtual functions + inheritance
Benefits Flexible, reusable, extensible code

‚úÖ What is Inheritance?

Inheritance is a fundamental concept in OOP that allows a class (called derived class or child class) to inherit properties and behaviors (attributes and methods) from another class (called base class or parent class).

  • Enables code reuse, reducing duplication.
  • Supports the ‚Äúis-a‚Äù relationship, meaning the derived class is a type of the base class.
  • Forms the foundation for polymorphism and hierarchical class structures.

üß† Key Concepts of Inheritance

Concept Description
Base Class The class whose members are inherited
Derived Class The class that inherits members from base class
Access Specifiers Determine which members are accessible in derived class (public, protected, private)
Single Inheritance Derived class inherits from one base class
Multiple Inheritance Derived class inherits from more than one base class
Multilevel Inheritance Inheritance chain across multiple levels
Hierarchical Inheritance Multiple derived classes inherit from the same base class
Reusability Inherited methods and attributes can be used without redefining
Overriding Derived class can redefine base class methods

üìå Syntax

class Base {
public:
    void display() {
        cout << "Base class function" << endl;
    }
};

// Single inheritance
class Derived : public Base {
public:
    void show() {
        cout << "Derived class function" << endl;
    }
};
  • public Base: public inheritance, base class public members remain public in derived class.
  • protected Base: public and protected members of base become protected in derived class.
  • private Base: public and protected members of base become private in derived class.

üßæ Example ‚Äì Single Inheritance 

#include <iostream>
using namespace std;

class Vehicle {
public:
    string brand;
    void honk() {
        cout << "Beep beep!" << endl;
    }
};

class Car : public Vehicle {
public:
    string model;
    void showInfo() {
        cout << "Brand: " << brand << ", Model: " << model << endl;
    }
};

int main() {
    Car car1;
    car1.brand = "Toyota"; // Inherited from Vehicle
    car1.model = "Corolla";

    car1.honk();           // Inherited method
    car1.showInfo();       // Derived class method
}

Output:

Beep beep!
Brand: Toyota, Model: Corolla

üîÑ Types of Inheritance

  1. Single Inheritance ‚Äì One base, one derived
  2. Multiple Inheritance ‚Äì One derived inherits from multiple bases 
class Base1 { public: void func1() {} };
class Base2 { public: void func2() {} };

class Derived : public Base1, public Base2 {}; // Multiple inheritance
  1. Multilevel Inheritance ‚Äì Chain of inheritance 
class A { public: void fA() {} };
class B : public A { public: void fB() {} };
class C : public B { public: void fC() {} };
  1. Hierarchical Inheritance ‚Äì One base, multiple derived 
class Base { };
class D1 : public Base { };
class D2 : public Base { };

‚ö†Ô∏è Important Rules

  1. Private members of the base class cannot be accessed directly by the derived class. Use public/protected getters/setters if needed.
  2. Constructors of base class are not inherited, but the base constructor runs before the derived constructor.
  3. Destructors run in reverse order: derived ‚Üí base.
  4. Multiple inheritance can lead to the diamond problem; solved using virtual inheritance.
  5. Overriding base class methods in derived classes enables runtime polymorphism.

‚úÖ What is Encapsulation?

Encapsulation is the concept of bundling data (attributes) and methods (functions) that operate on the data into a single unit, typically a class.

  • It hides the internal state of an object and exposes a controlled interface to interact with it.
  • Promotes data security, integrity, and maintainability.
  • Often referred to as data hiding in OOP.

Key idea:

  • Private members store data.
  • Public methods allow controlled access and modification.

üß† Key Concepts of Encapsulation

Concept Description
Data Hiding Keep internal variables private to prevent unauthorized access
Accessors / Getters Public methods that allow reading private data
Mutators / Setters Public methods that allow modifying private data safely
Class Interface Public methods define the only way to interact with the object
Benefits Improves security, reduces errors, makes code easier to maintain and extend

üìå Syntax

class BankAccount {
private:
    double balance; // Hidden from outside

public:
    // Constructor
    BankAccount(double initialBalance) {
        if (initialBalance >= 0)
            balance = initialBalance;
        else
            balance = 0;
    }

    // Getter
    double getBalance() {
        return balance;
    }

    // Setter
    void deposit(double amount) {
        if (amount > 0)
            balance += amount;
    }

    void withdraw(double amount) {
        if (amount > 0 && amount <= balance)
            balance -= amount;
        else
            cout << "Insufficient funds or invalid amount" << endl;
    }
};

üßæ Example Program 

#include <iostream>
using namespace std;

int main() {
    BankAccount account(100); // Initial balance: 100

    cout << "Initial balance: " << account.getBalance() << endl;

    account.deposit(50);
    cout << "After deposit: " << account.getBalance() << endl;

    account.withdraw(30);
    cout << "After withdrawal: " << account.getBalance() << endl;

    account.withdraw(200); // Invalid withdrawal
    cout << "After failed withdrawal: " << account.getBalance() << endl;

    return 0;
}

Output:

Initial balance: 100
After deposit: 150
After withdrawal: 120
Insufficient funds or invalid amount
After failed withdrawal: 120

üîÑ How Encapsulation Works in the Example

  1. balance is private, so it cannot be modified directly from outside the class.
  2. Public methods (deposit, withdraw, getBalance) provide controlled access.
  3. The class enforces rules, such as preventing negative balances or invalid deposits.
  4. Users of the class don‚Äôt need to know the internal logic, only the public interface.

‚ö†Ô∏è Important Rules

  1. Always keep attributes private or protected unless there is a strong reason to make them public.
  2. Use getter/setter methods to allow controlled access.
  3. Encapsulation helps prevent unintended interference with object data.
  4. It is closely related to abstraction, as both hide internal implementation details.
  5. Encapsulation improves maintainability, debugging, and security of programs.
