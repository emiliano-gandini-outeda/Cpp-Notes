🟦 Abstract Data Types (ADTs)

✅ What is an Abstract Data Type?

An Abstract Data Type (ADT) is a mathematical model for a data structure that defines:

  1. A set of values the type can hold
  2. A set of operations that can be performed on the values

Key idea:

  • ADTs define what operations do, but not how they are implemented.
  • They provide an interface that hides implementation details.

Common ADTs:

  • Stack – Last-In, First-Out (LIFO)
  • Queue – First-In, First-Out (FIFO)
  • List – Ordered collection of elements
  • Priority Queue – Queue with element priorities
  • Deque – Double-ended queue
  • Graph – Set of vertices and edges

🧠 Key Concepts of ADTs

Concept Description
Data The values stored in the ADT
Operations Functions allowed on the ADT (push, pop, insert, delete, search, etc.)
Interface Defines operations without specifying implementation
Encapsulation Implementation details are hidden from the user
Flexibility Same ADT can have multiple implementations (arrays, linked lists, trees)
Benefits Modularity, maintainability, reusable code, implementation independence

📌 ADT vs Data Structure

Aspect Abstract Data Type (ADT) Data Structure
Focus What operations are allowed How data is stored
Example Stack with push/pop/top Stack implemented using array or linked list
Implementation Hidden Explicit in code
Purpose Interface Performance and memory optimization

🧾 Example – Stack ADT

Stack ADT Operations:

  1. push(element) – Add an element to the top
  2. pop() – Remove the top element
  3. top() / peek() – Access the top element without removing
  4. isEmpty() – Check if the stack is empty

Implementation using C++ class (Array-based): 

#include <iostream>
using namespace std;

class Stack {
private:
    int arr[100];
    int topIndex;

public:
    Stack() { topIndex = -1; }

    void push(int x) {
        if (topIndex < 99) arr[++topIndex] = x;
        else cout << "Stack overflow!" << endl;
    }

    void pop() {
        if (topIndex >= 0) topIndex--;
        else cout << "Stack underflow!" << endl;
    }

    int top() {
        if (topIndex >= 0) return arr[topIndex];
        else return -1; // Empty stack
    }

    bool isEmpty() {
        return topIndex == -1;
    }
};

int main() {
    Stack s;
    s.push(10);
    s.push(20);
    cout << "Top element: " << s.top() << endl;
    s.pop();
    cout << "Top after pop: " << s.top() << endl;
}

Output:

Top element: 20
Top after pop: 10

🔄 Key Points from Example

  1. User interacts only via the interface (push, pop, top, isEmpty).
  2. Internal array and topIndex are hidden (encapsulation).
  3. Same Stack ADT can be implemented using a linked list instead of an array.
  4. ADTs ensure modularity: code using Stack doesn’t need to know its internal implementation.

⚠️ Important Rules

  1. ADTs define behavior, not implementation.
  2. Operations must maintain the ADT’s invariants (e.g., LIFO for stack, FIFO for queue).
  3. Multiple data structures can implement the same ADT differently depending on performance needs.
  4. Using ADTs allows clean interfaces, making code easier to maintain, test, and extend.
  5. ADTs are language-independent concepts, though implemented in languages like C++, Java, or Python.
