‚úÖ What is Binary Search?

Binary Search is an efficient search algorithm that finds the position of a target element in a sorted array.

  • It works by repeatedly dividing the search interval in half.
  • Compare the target with the middle element of the current interval:
    • If equal ‚Üí found
    • If target < middle ‚Üí search left half
    • If target > middle ‚Üí search right half
  • Requires the array to be sorted in ascending (or descending) order.

üìå How Binary Search Works

  1. Initialize low = 0 and high = n - 1.
  2. Find mid = (low + high) / 2.
  3. Compare arr[mid] with the target:
    • If equal ‚Üí return mid
    • If target < arr[mid] ‚Üí search left subarray
    • If target > arr[mid] ‚Üí search right subarray
  4. Repeat until low > high.
  5. If the element is not found, return -1.

üßæ Example ‚Äì Iterative Binary Search 

#include <iostream>
using namespace std;

int binarySearch(int arr[], int n, int target) {
    int low = 0, high = n - 1;

    while (low <= high) {
        int mid = low + (high - low) / 2;

        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) low = mid + 1;
        else high = mid - 1;
    }

    return -1; // Not found
}

int main() {
    int arr[] = {1, 3, 5, 7, 9, 11};
    int n = sizeof(arr) / sizeof(arr[0]);
    int target = 7;

    int index = binarySearch(arr, n, target);

    if (index != -1)
        cout << "Element found at index: " << index << endl;
    else
        cout << "Element not found" << endl;

    return 0;
}

Output:

Element found at index: 3

üîÑ Step-by-Step Example

Array: [1, 3, 5, 7, 9, 11], Target = 7

  1. low = 0, high = 5 ‚Üí mid = 2 ‚Üí arr[2] = 5 ‚Üí 7 > 5 ‚Üí search right
  2. low = 3, high = 5 ‚Üí mid = 4 ‚Üí arr[4] = 9 ‚Üí 7 < 9 ‚Üí search left
  3. low = 3, high = 3 ‚Üí mid = 3 ‚Üí arr[3] = 7 ‚Üí found!

‚ö†Ô∏è Important Rules

  1. Array must be sorted before using Binary Search.
  2. Time complexity:
    • Best-case: O(1) ‚Üí target is middle element
    • Worst-case: O(log n) ‚Üí repeatedly halving search space
    • Average-case: O(log n)
  3. Space complexity:
    • Iterative: O(1)
    • Recursive: O(log n) due to call stack
  4. Faster than Linear Search for large arrays.
  5. Not stable or applicable for unsorted data.
