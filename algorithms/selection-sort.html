🟨 Selection Sort

✅ What is Selection Sort?

Selection Sort is a simple comparison-based sorting algorithm.

  • It repeatedly finds the minimum element from the unsorted part of the array.
  • Then it swaps that element with the first element of the unsorted part.
  • After each iteration, the sorted portion grows from left to right.

🧠 Key Concepts

Concept Description
Comparison-based Sorts by finding the minimum element repeatedly
Partition Divides the array into sorted and unsorted parts
Swap Puts the smallest element of the unsorted part into its correct place
Not stable Equal elements may change their relative order
In-place Uses O(1) extra memory
Time complexity Always O(n²), regardless of input
Best for Small arrays when swaps are expensive

📌 How Selection Sort Works

  1. Start with the first position.
  2. Find the minimum element in the unsorted portion.
  3. Swap it with the element at the current position.
  4. Move to the next position.
  5. Repeat until the array is fully sorted.

🧾 Example – Standard Selection Sort 

#include <iostream>
using namespace std;

void selectionSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int minIndex = i;

        // Find the minimum element in unsorted portion
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIndex])
                minIndex = j;
        }

        // Swap the found minimum with the first element
        swap(arr[i], arr[minIndex]);
    }
}

int main() {
    int arr[] = {64, 25, 12, 22, 11};
    int n = sizeof(arr) / sizeof(arr[0]);

    selectionSort(arr, n);

    cout << "Sorted array: ";
    for (int i = 0; i < n; i++) cout << arr[i] << " ";
    cout << endl;

    return 0;
}

Output:

Sorted array: 11 12 22 25 64

🔄 Step-by-Step Explanation

Array = [64, 25, 12, 22, 11]

  1. Find min in [64,25,12,22,11] → 11 → Swap with 64 → [11,25,12,22,64]
  2. Find min in [25,12,22,64] → 12 → Swap with 25 → [11,12,25,22,64]
  3. Find min in [25,22,64] → 22 → Swap with 25 → [11,12,22,25,64]
  4. Find min in [25,64] → 25 → Already in place → [11,12,22,25,64]

Sorted! ✅

⚠️ Important Rules

  1. Not stable → equal elements may swap order.
  2. In-place → needs no extra memory.

  3. Time complexity:

    • Best-case: O(n²)
    • Worst-case: O(n²)
    • Average-case: O(n²) → Always quadratic, no shortcuts.
  4. Number of swaps is minimal: at most n-1 swaps, even if many comparisons are made.
  5. Works better than Bubble Sort when swaps are expensive, because Bubble Sort does many unnecessary swaps.
