‚úÖ What is Big O Notation?

Big O notation is a mathematical way to describe the performance and efficiency of an algorithm:

  • Measures how runtime or memory usage grows relative to the input size n.
  • Focuses on the worst-case scenario, although it can also describe average or best cases.
  • Ignores constant factors and lower-order terms, emphasizing growth trends.

Big O helps us:

  • Predict algorithm efficiency for large inputs
  • Compare algorithms to choose the best one
  • Identify potential bottlenecks

üß† Key Concepts

Concept Description
Time Complexity How the runtime grows with input size n
Space Complexity How memory usage grows with input size n
Worst-case Maximum number of operations the algorithm could take
Average-case Expected number of operations for typical input
Best-case Minimum number of operations possible
Constants & lower-order terms Ignored in Big O (e.g., O(2n) ‚Üí O(n))

4Ô∏è‚É£ O(log n) ‚Äì Logarithmic Time

Binary search halves the search space each step: 

#include <iostream>
using namespace std;

int binarySearch(int arr[], int n, int target) {
    int left = 0, right = n - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}

int main() {
    int arr[7] = {1,3,5,7,9,11,13};
    cout << binarySearch(arr, 7, 7) << endl;
    return 0;
}

‚ö†Ô∏è Important Rules and Tips

  1. Ignore constants: O(2n) = O(n).
  2. Ignore lower-order terms: O(n¬≤ + n) = O(n¬≤).
  3. Focus on growth trend, not exact runtime.
  4. Different operations have different complexities (array access O(1), loop O(n), nested loop O(n¬≤)).
  5. Space complexity can also be expressed in Big O (e.g., recursion ‚Üí call stack O(n)).
  6. Use Big O to compare algorithms for large inputs ‚Äî constants matter less as n grows.
