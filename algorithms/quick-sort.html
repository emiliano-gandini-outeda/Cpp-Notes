🟪 Quick Sort

✅ What is Quick Sort?

Quick Sort is a divide and conquer sorting algorithm.

  • It works by selecting a pivot element and partitioning the array into two subarrays:
    1. Elements less than or equal to the pivot
    2. Elements greater than the pivot
  • Recursively applies the same process to the subarrays.
  • Efficient for large datasets, often faster in practice than Merge Sort because it does not require extra memory.

🧠 Key Concepts

Concept Description
Divide and Conquer Splits the array around a pivot element
Partitioning Reorders elements so that left < pivot < right
Recursive Sorts subarrays recursively
In-place Usually does not require extra memory
Not stable Equal elements may change relative order
Time complexity Average: O(n log n), Worst-case: O(n²)
Best for Large arrays, practical general-purpose sorting

📌 How Quick Sort Works

  1. Pick a pivot element (first, last, random, or median).
  2. Partition the array around the pivot.
  3. Recursively apply Quick Sort to the left and right subarrays.
  4. Combine results (no merging needed since partitioning ensures order).

🧾 Example – Standard Quick Sort 

#include <iostream>
using namespace std;

// Partition the array
int partition(int arr[], int low, int high) {
    int pivot = arr[high]; // Choose last element as pivot
    int i = low - 1;

    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            swap(arr[i], arr[j]);
        }
    }
    swap(arr[i + 1], arr[high]);
    return i + 1;
}

// Recursive quick sort
void quickSort(int arr[], int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}

int main() {
    int arr[] = {10, 7, 8, 9, 1, 5};
    int n = sizeof(arr) / sizeof(arr[0]);

    quickSort(arr, 0, n - 1);

    cout << "Sorted array: ";
    for (int i = 0; i < n; i++) cout << arr[i] << " ";
    cout << endl;

    return 0;
}

Output:

Sorted array: 1 5 7 8 9 10

🔄 Step-by-Step Example

Array: [10, 7, 8, 9, 1, 5]

  1. Pivot = 5 → Partition → [1, 5, 8, 9, 10, 7]
  2. Left subarray [1] → Already sorted
  3. Right subarray [8, 9, 10, 7] → Pivot = 7 → Partition → [7, 8, 9, 10]
  4. Left [] → Empty
  5. Right [8, 9, 10] → Pivot = 10 → Partition → [8, 9, 10]
  6. Fully sorted array → [1, 5, 7, 8, 9, 10]

⚠️ Important Rules

  1. Not stable → equal elements may change relative order.
  2. In-place → does not require extra memory for merging.

  3. Time complexity:

    • Best-case: O(n log n) → balanced partitioning
    • Average-case: O(n log n) → typical random data
    • Worst-case: O(n²) → already sorted or reversed array (can be improved with random pivots)
  4. Space complexity: O(log n) due to recursion stack.
  5. Quick Sort is usually faster than Merge Sort in practice for arrays.
  6. Choosing a good pivot is key to efficiency.
