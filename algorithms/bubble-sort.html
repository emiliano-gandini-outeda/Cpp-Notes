‚úÖ What is Bubble Sort?

Bubble Sort is a simple comparison-based sorting algorithm.

  • It repeatedly compares adjacent elements and swaps them if they are in the wrong order.
  • This ‚Äúbubbling‚Äù process continues until the entire array is sorted.
  • Easy to understand and implement but inefficient for large datasets.

üìå How Bubble Sort Works

  1. Start from the first element of the array.
  2. Compare it with the next element.
  3. If they are in the wrong order, swap them.
  4. Move to the next pair and repeat until the end of the array.
  5. After each pass, the largest element moves to its correct position.
  6. Repeat for all remaining elements until the array is sorted.

üßæ Example ‚Äì Standard Bubble Sort 

#include <iostream>
using namespace std;

void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n-1; i++) {           
        for (int j = 0; j < n-i-1; j++) {     
            if (arr[j] > arr[j+1]) {
                swap(arr[j], arr[j+1]);       
            }
        }
    }
}

int main() {
    int arr[] = {64, 34, 25, 12, 22, 11, 90};
    int n = sizeof(arr)/sizeof(arr[0]);
    bubbleSort(arr, n);
    cout << "Sorted array: ";
    for (int i = 0; i < n; i++) cout << arr[i] << " ";
    cout << endl;
    return 0;
}

Output: Sorted array: 11 12 22 25 34 64 90

üîÑ Example ‚Äì Optimized Bubble Sort

Optimization: Stop if no swaps are made during a pass ‚Üí array is already sorted. 

#include <iostream>
using namespace std;

void optimizedBubbleSort(int arr[], int n) {
    bool swapped;
    for (int i = 0; i < n-1; i++) {
        swapped = false;
        for (int j = 0; j < n-i-1; j++) {
            if (arr[j] > arr[j+1]) {
                swap(arr[j], arr[j+1]);
                swapped = true;
            }
        }
        if (!swapped) break;
    }
}

int main() {
    int arr[] = {11, 12, 22, 25, 34, 64, 90};
    int n = sizeof(arr)/sizeof(arr[0]);
    optimizedBubbleSort(arr, n);
    cout << "Sorted array: ";
    for (int i = 0; i < n; i++) cout << arr[i] << " ";
    cout << endl;
    return 0;
}
  • Explanation: Best-case complexity becomes O(n) if the array is already sorted.

‚ö†Ô∏è Important Rules

  1. In-place sorting ‚Üí does not require extra memory.
  2. Stable algorithm ‚Üí equal elements maintain relative order.

  3. Time complexity:

    • Worst-case: O(n¬≤) ‚Üí reversed array
    • Best-case: O(n) ‚Üí already sorted (with optimization)
    • Average-case: O(n¬≤)
  4. Not suitable for large arrays ‚Üí inefficient compared to quicksort, mergesort, or heapsort.
  5. Can be optimized using early exit or by reducing the inner loop boundary.
