🌳 Depth-First Search (DFS)

✅ What is Depth-First Search?

Depth-First Search (DFS) is a graph traversal algorithm that explores as far as possible along each branch before backtracking.

  • It starts at a source node, then visits an adjacent unvisited node recursively.
  • If a node has no unvisited neighbors, DFS backtracks to the previous node.
  • Can be implemented recursively or using a stack.
  • Works for both directed and undirected graphs.

🧠 Key Concepts

Concept Description
Traversal type Explores graph depth-first
Graph type Directed or undirected
Implementation Recursive or stack-based
Visited tracking Keeps track of visited nodes to avoid cycles
Time complexity O(V + E) → V = vertices, E = edges
Space complexity O(V) for recursion stack or explicit stack
Use cases Detect cycles, connectivity, topological sort, path finding

📌 How DFS Works

  1. Start at the source node.
  2. Mark the node as visited.
  3. For each adjacent unvisited node, perform DFS recursively.
  4. Backtrack when all neighbors are visited.
  5. Repeat until all reachable nodes are visited.

🧾 Example – DFS using adjacency list 

#include <iostream>
#include <vector>
using namespace std;

void dfs(int node, vector<vector<int>>& graph, vector<bool>& visited) {
    visited[node] = true;
    cout << node << " ";

    for (int neighbor : graph[node]) {
        if (!visited[neighbor])
            dfs(neighbor, graph, visited);
    }
}

int main() {
    int vertices = 5;
    vector<vector<int>> graph(vertices);

    // Add edges
    graph[0] = {1, 2};
    graph[1] = {2};
    graph[2] = {0, 3};
    graph[3] = {3, 4};

    vector<bool> visited(vertices, false);

    cout << "DFS traversal starting from node 0: ";
    dfs(0, graph, visited);
    cout << endl;

    return 0;
}

Output (one possible traversal):

DFS traversal starting from node 0: 0 1 2 3 4

🔄 Step-by-Step Example

Graph (edges):

0 → 1, 0 → 2, 1 → 2, 2 → 0, 2 → 3, 3 → 3, 3 → 4

DFS starting at node 0:

  1. Visit 0 → mark visited
  2. Go to 1 → mark visited
  3. 1’s neighbor 2 → visit 2
  4. 2’s neighbors 0 (visited) and 3 → visit 3
  5. 3’s neighbors 3 (visited) and 4 → visit 4
  6. All nodes reachable from 0 are visited → finish

⚠️ Important Rules

  1. DFS may not find the shortest path in weighted or unweighted graphs.
  2. Requires visited tracking to avoid infinite loops in cyclic graphs.
  3. Time complexity: O(V + E) → visits each vertex and edge once.
  4. Space complexity: O(V) for recursion stack (or explicit stack in iterative DFS).
  5. Can be used for:
    • Detecting cycles in a graph
    • Topological sorting
    • Pathfinding in mazes
    • Solving puzzles (e.g., Sudoku, N-Queens)
