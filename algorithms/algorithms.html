Algorithms are step-by-step procedures for solving problems efficiently. Key concepts include recursion, complexity analysis, and sorting algorithms.

üîÅ Recursion

  • Recursion is a technique where a function calls itself to solve a problem by breaking it into smaller subproblems.

  • Key components:

    • Base case: Stops recursion, returning a direct result.
    • Recursive case: Calls the function with a simpler or smaller input, moving toward the base case.
  • Uses: factorials, binary search, merge sort, generating permutations, tree traversal.
  • ‚ö†Ô∏è Careful design is crucial; missing or incorrect base cases can cause infinite loops or stack overflow.

  • Big O notation describes asymptotic behavior as input size grows, ignoring constants and lower-order terms.

üóÇÔ∏è Sorting Algorithms

Sorting algorithms reorder data based on a criterion (usually ascending/descending). They are evaluated by time and space complexity.

Basic Sorting Algorithms

Useful for learning or small datasets; not optimal for large datasets.

üíß Bubble Sort: Compares adjacent elements and swaps them if out of order; repeats until sorted.

  • Complexity: O(n¬≤)
  • Best for small or nearly sorted lists.

üìù Insertion Sort: Builds a sorted section gradually by inserting each element in the correct position relative to the already sorted elements.

  • Complexity: Average/Worst O(n¬≤), Best O(n)
  • Efficient for small or partially sorted datasets.

üîé Selection Sort: Finds the smallest element and places it at the beginning, repeating for the rest of the list.

  • Complexity: O(n¬≤) in all cases
  • Simple, but not efficient for large datasets.

üîç Search Algorithms

Search algorithms are used to find elements within a dataset. Efficiency depends on the data structure and algorithm choice.

üîπ Linear Search: Checks each element sequentially until the target is found.

  • Complexity: O(n)
  • Simple but inefficient for large datasets.

üîπ Binary Search: Repeatedly divides a sorted dataset in half to locate the target element.

  • Complexity: O(log n)
  • Efficient but requires the data to be sorted.
